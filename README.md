# Assignment 1 : TypeScript Blog

1. What are some differences between interfaces and Types in TypeScript?
   Interface এবং Types এর মধ্যে মূল পার্থক্য হচ্ছে, Type অনেক বেশি ফ্লেক্সিবল। Type দিয়ে Object ছাড়াও String, Number, Union এমনকি Tuple ও ডিফাইন করা যায়। মানে Type Primitive এবং custom Union সাপোর্ট করে। একই নামের Type একবারের বেশি লিখলে এরর দেয়।
   অন্যদিকে Interface শুধুমাত্র Object এর জন্য স্ট্রাকচার ডিফাইন করতে পারে। Interface দিয়ে অন্য কোনো ডেটা টাইপ ডিফাইন করা যায় না। একই নামের ইন্টারফেস একধিকবার লিখলেও একেবারে merge হয়ে যায়।

2. Explain the differences between any, unknown and never types in TypeScript.
   any: any ব্যবহার করার অর্থ হলো এখানে কোনো টাইপ চেকিং ই হবে না। এখানে যা খুশি তাই হতে পারে। যেটা সেইফ না। এটাকে টাইপস্ক্রিপ্টের “Escape Hatch” ও বলা হয়। আমরা যখন কোনো ভ্যারিয়েবল টাইপে any দিই, তখন টাইপস্ক্রিপ্ট এই ভ্যারিয়েবলের জন্য সবরকম টাইপ চেকিং বন্ধ করে দেয়। এটা অনেকটা জাভাস্ক্রিপ্ট কোডের মতো হয়ে যায়। মানে, any লিখলে কিছু যায় আসে না। তখন এটাকে জাভাস্ক্রিপ্টের কোড বললেও ভুল কিছু হবে না।
   তবে any ব্যবহার করলে কোড লিখার সময় এরর না দিলেও রান করার সময় কোড ক্র্যাশ করতে পারে। যেমনঃ

```
let var : any = 10;
console.log(var.toUpperCase());
```

ব্যাখ্যাঃ শুরুতে ভ্যারিয়েবল টা নাম্বার দিয়ে ডিফাইন করলাম। কিন্তু প্রিন্ট করার সময় সেটাকে ধরে তার উপর স্ট্রিং এর একটা ফাংশন দিয়ে কাজ করতে চাইলাম। এখানে any লিখার জন্য এমন মনে হতে পারে, এটা যেকোনো টাইপেই কাজ করবে। কিন্তু রান টাইমে এরর দিবে। কারণ, টাইপস্ক্রিপ্ট এই ভ্যারিয়েবল টাকে একটা নাম্বার হিসেবে চিনবে। তাই নাম্বারের উপর স্ট্রিং এর বিল্ট ইন ফাংশন কাজ করবে না।

unknown: এটা any এর তুলনায় অনেক সেইফ। এটা বোঝাবে যে টাইপ টা জানা নেই। যা দিয়েই ডিফাইন করবো, সেটাই ধরে নিবে। মানে, টাইপস্ক্রিপ্টের কাছে জিনিসটা আপাতত জানা নেই। কিন্তু ভ্যালু অ্যাসাইন করার পর সে বুঝে নিতে পারবে।
unknown এর একটা ভালো দিক হলো, এটা যেকোনো ভ্যালু ই গ্রহণ করতে পারবে। কিন্তু টাইপ চেক না করা পর্যন্ত সেই ভ্যালু আমি ব্যবহার করতে পারবো না। টাইপ চেক করার পর ই আমি কেবল ভ্যালু টা ব্যবহার করতে পারবো। যেমনঃ

```
let var: unknown = “Good Morning”;
// console.log(var.toUpperCase()); // এটা করতে দিবে না। টাইপ চেক করতে হবে।
if(typeof var === “string”) {
	console.log(var.toUpperCase()); // এখন পারফেক্টলি কাজ করবে।
}
```

never: কখনোই ঘটবে না বা কিছু রিটার্ন করবে না এমন ক্ষেত্রে আমরা never ব্যবহার করি। এমনকি যে ফাংশন কখনোই শেষ হবে না তার রিটার্ন টাইপেও never ব্যবহার করা হয়। never এর ভালো দুইটি use-case আছে।

i) এমন কোনো ফাংশন , যেটা সবসময়ই কেবল এরর থ্রো করে। এই ধরনের ফাংশন যেন কখনোই সফলভাবে রিটার্ন না করতে পারে তার জন্য আমরা never ব্যবহার করি। যেমনঃ

```
	function throwError(message: string): never {
		Throw new Error(message);
}
```

ii) ইনফিনিটি লুপ বিশিষ্ট কোনো ফাংশনের কোড যাতে তার স্কোপ থেকে বের হতে না পারে তার জন্যও আমরা never ব্যবহার করি। যেমনঃ

```
    function infiniteLoop(): never {
	while(true) {
			console.log(“Running infinite time”);
            }
    }
```
